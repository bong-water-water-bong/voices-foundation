# Contributing

There are two very different kinds of contributors here.

## Language contributors

Speakers, learners, linguists, community members. **The most important contributors in this project.** Your contribution is governed by the community you are part of, not by GitHub PR review. See `GOVERNANCE.md`.

If you are interested in contributing in this way, the right entry point is **your community's Language Authority**, not this repo.

## Code contributors

Engineers contributing to the framework. Your work is governed by normal open-source norms (PRs, review, tests) **plus** the rule that no PR may relax governance constraints.

### What we will accept
- Bug fixes
- Performance improvements
- New backends / hardware support
- Better tokenizer implementations
- Tests, documentation, examples
- Tools that make recording, transcription, or training easier for non-technical community members

### What we will not accept
- Code that strips speaker attribution
- Code that enables training on restricted content
- Vendor lock-in to any hosted service
- Telemetry that phones home without explicit community opt-in
- Features whose primary use case is aggregating cross-community data

## A note on motivation

If you are here because you want to "save" Indigenous languages: don't. Communities don't need saving. They need tools they own and control. If you can build that, you are welcome. If you need to be the protagonist of the story, this is not the project for you.
