# Agent Handoff & Orchestration

This wiki page defines the agent handoff and orchestration pattern to be used by automation agents interacting with this repository.

Roles:
- Coordinator agent: assigns tasks to worker agents, tracks progress, updates this wiki.
- Worker agents: run checks, CI, scans, and produce artifacts; report status back to Coordinator.

Handoff protocol:
1. Coordinator creates a task entry with: id, intent, required permissions, inputs, expected outputs, and timeout.
2. Worker claims task and updates the task state (claimed, running, completed, failed) with timestamps.
3. On completion, worker attaches artifacts and a concise summary; Coordinator reviews and marks task done.

Files/Pages autogenerated by agents:
- /Tasks (table of active tasks)
- /Artifacts (links to logs, reports)

Update policy:
- Agents must only modify their task entries and artifact links.
- Human maintainers may edit this page to refine orchestration rules.

Safety:
- Agents must not commit secrets to the repo or wiki.
- Low-confidence actions require human approval.

Example task template:
```
- id: task-2026-05-24-001
  intent: run-ci-and-collect-failures
  repo: owner/repo
  inputs: [branch: main]
  outputs: [workflow-report.md]
  status: pending
```
